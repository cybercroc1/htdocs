Редактирование списков значений (вариантов ответа).

--Значение - Элемент анкеты 4 уровня.

--Список значенй (варианты ответа).
Список предустановленных значений для выбора. Задается для объектов 3 уровня:
Вопрос - выбор
Вопрос - радио
Вопрос - множ. выбор
Вопрос - галочки

--Аттрибуты варианта ответа.
--"Текстовое значение" - задает текст варианта ответа, который будет выбирать оператор, он же отображается в отчете. Не может быть пустым.
--"Кодовое значение" - задает кодовое значение ответа в отчете (если выбрана соответствующая опция). Может быть пустым и не уникальным.
--"Ключ квоты" - по этому ключу создаются квоты по вопросам, если ключ квоты пустой, квота создана не будет.
--"Вторичный ключ" - по этому ключу можно создать зависимость от предыдущих вопросов. Может быть пустым и не уникальным.

--Редактирование списка значений.
На странице "Редактирование анкеты" нажмите на "карандашек " напротив объекта, для которого хотите задать список значений.
В правом фрейме откроется страница редактирования списков значений.

--Вставка значений из буфера обмена
1. Скопируйте в буфер обмена, например из таблицы EXCEL столбцы, содержащие данные для полей "Текстовое значение", "Кодовое значение", "Ключ квоты", "Вторичный ключ" (можно скопировать 1, 2, 3 или 4 столбца).
2. Вставьте скопированные данные в окошко с надписью "буфер обмена".
3. Зажмите CTRL и нажмите синий "+" в той строке, после которой хотите добавить данные из буфера.
Для браузера Internet Explorer 2-й шаг можно пропустить ( Это связано с тем, что с недавнего времени все браузеры, кроме IE запретили прямой доступ к буферу обмена для предотвращения молчеливого воровства паролей из буфера обмена :)
Внимание! При вставке разделителем полей является символ табуляции, а разделителем строк символы перевода строки (именно в таком виде копируется в буфер таблица из EXCEL), поэтому проследите, что бы при копировании из таблицы внутри ячеек небыло этих символов.

--Ротация (сортировка) вариантов ответа.
 Тип ротации назначается объекту(вопросу) соответствующего типа на странице "Редактирование анкеты".
Можно установить 2 типа ротации вариантов ответа:
По порядку - варианты ответа будут отображаться в том порядке, в котором они отсортированы на странице редактирования списка значений.
Случайно - варианты ответа будут отображаться в случайном порядке.
При этом на странице редактирования списка значений каждому варианту ответа можно задать свойство "Всегда внизу ", варианты ответа, имеющие свойство "Всегда внизу" будут отображаться внизу списка не зависимо от типа ротации. Между собой такие варианты ответа будут отсортированы так, как на странице редактирования списка значений. Это нужно, например для того, что бы задать вариант(ы) ответа "Другое", и он(и) всегда был(и) внизу списка.

--Вариант ответа "Другое".
Для того, что бы назначить вариант ответа "Другое", нужно на странице редактирования списка значений задать варианту(ам) ответа количество дополнительных полей, отлиное от нуля. Тогда, если при анкетировании будет выбран этот вариант(ы) ответа, то под ним(и) отобразится(ятся) дополнительное(ые) поле(я) для ввода текста, в заданном количестве.

-- "Влияет на"
Вопросы с одиночным выбором можно связать по ключу квоты с исходным не уникальным полем, это значит, что при выборе варианта ответа, значение связанного исходного поля поменяется на ключ квоты выбранного варианта ответа, а так же, если исходное поле квотируемое или индексируемое пересчитается статиcтика и квоты с учетом сделанных изменений.
Данное изменение исходного поля не повлияет на прохождение текущей анкеты, даже если в результате изменения превысилась квота по исходным полям. Влияние не работает, если ключ квоты пустой.
Если в качестве целевого поля выбраны такие особые поля, как "Телефон" или "Часовой пояс", поменяется только значение исходных данных, но связанные с записью телефон или чп не изменятся.

-- "Зависит от"
Для всех вопросов с выбором значений можно установить зависимость отображаемого оператору списка ответов от значения выбранного квотируемого или индексируемого исходного поля или от любого предыдущего вопроса. Это значит, что оператору будут предложены на выбор только те варинты ответа, вторичный ключ которых равен значению исходного поля или ключу квоты (первичному ключу) влияющего вопроса. Зависимость не работает, если ключ квоты влияющего вопроса или вторичный ключ зависимого вопроса пустой.
